# DSA: 3Sum (Sort + Two Pointers)

Return all unique triplets in `nums` that sum to `0`.

**LeetCode:** [3Sum](https://leetcode.com/problems/3sum/)

---

## Why it matters for FE

Fix one element + two pointers is a common multi-constraint filter pattern.

---

## Approaches & trade-offs

| Approach | Time | Space | When to use / skip |
|----------|------|-------|---------------------|
| Triple nested loops | O(n³) | O(1) | Skip |
| **Sort + fix i + two pointers** | **O(n²)** | **O(1)** | **Default** |

---

## Algorithm

1. Sort `numbers` ascending.
2. For each index `i`:
   - Skip if `numbers[i]` equals previous (duplicate triplets).
   - If `numbers[i] > 0`, break (no negative sum possible).
   - `left = i + 1`, `right = n - 1`.
   - While `left < right`:
     - If sum is 0: push triplet; skip duplicate `left` and `right`; move both.
     - If sum < 0: `left++`; else `right--`.
3. Return `result`.

---

## JavaScript solution

```javascript
function threeSum(numbers) {
  numbers.sort((a, b) => a - b);
  const result = [];

  for (let i = 0; i < numbers.length; i++) {
    if (i > 0 && numbers[i] === numbers[i - 1]) continue;
    if (numbers[i] > 0) break;

    let left = i + 1;
    let right = numbers.length - 1;

    while (left < right) {
      const sum = numbers[i] + numbers[left] + numbers[right];

      if (sum === 0) {
        result.push([numbers[i], numbers[left], numbers[right]]);
        left++;
        right--;
        while (left < right && numbers[left] === numbers[left - 1]) left++;
        while (left < right && numbers[right] === numbers[right + 1]) right--;
      } else if (sum < 0) {
        left++;
      } else {
        right--;
      }
    }
  }

  return result;
}
```

## Big-O analysis

- **Time complexity:** O(n²) — outer loop × two pointers.
- **Space complexity:** O(1) extra (excluding output).

---

## Gotchas

- Sort first; skip duplicate `i`, `left`, `right` after a hit.
